## Описание проекта:

### Цель практики:
Cоздать программное обеспечение способное шифровать и расшифровывать информацию хранимую в файлах.
В основе шифрования должен быть заложен механизм проверки целостности (хеш-значение) передаваемых или хранимых данных.
Хеш-значение должно быть частью шифруемых данных с целью дальнейшего приобретения аутентифицирующего свойства.
В целях предотвращения криптоанализа, алгоритм шифрования должен быть надёжным,
должен использовать режим шифрования отличный от ECB (электронной кодовой книги),
а в основе вектора инициализации и/или уникального числа должен
использоваться криптографически стойкий генератора псевдослучайных чисел.

### Характеристики приложения:
-блочный алгоритм шифрования Speck со 128-битным блоком и 128-битным ключом;
-режим шифрования CBC Режим сцепления блоков шифротекста) со случайным IV (вектором инициализации)
 на базе выхода КСГПСЧ (криптографически стойкого генератора псевдослучайных чисел);
-хеш-функция на базе алгоритмов Speck и MDC-2;
-КСГПСЧ на базе шифра Speck и Fortuna подобном алгоритме;
